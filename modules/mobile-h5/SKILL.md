# Mobile H5 Module

## Purpose

Apply mobile-first H5 and responsive application rules.

## Rules

- Design for touch targets.
- Avoid hover-only interactions.
- Use safe-area handling.
- Prefer bottom sheets over desktop modals where appropriate.
- Avoid fixed pixel layouts.
- Prevent horizontal overflow.
- Consider keyboard overlap for forms.

## Review Checks

- small tap targets
- horizontal overflow
- viewport height issues
- iOS safe-area issues
- keyboard overlap
