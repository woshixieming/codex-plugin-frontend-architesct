# Component Patterns Reference

## Button

Action trigger.

Required states:

- default
- hover
- active
- focus
- disabled
- loading

## Input

User data entry.

Required:

- label
- validation
- error message
- help text when needed

## Dialog

Blocking overlay.

Required:

- focus trap
- escape handling
- restore focus
- accessible title

## DataTable

Structured data display.

Required:

- loading
- empty
- error
- pagination or virtualization where needed
- keyboard and screen reader considerations

## Empty State

No data feedback.

Required:

- clear message
- optional action
