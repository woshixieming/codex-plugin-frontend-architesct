# Module Authoring

## Module Structure

```text
modules/my-module/
  SKILL.md
```

## Required Sections

```markdown
# Module Name

## Purpose

## When to Use

## Rules

## Generation Rules

## Review Checks

## Anti-Patterns
```

## Guidelines

- Keep modules narrow.
- Do not duplicate core rules.
- Add only framework, library, or domain-specific guidance.
- Include examples only when they clarify behavior.
