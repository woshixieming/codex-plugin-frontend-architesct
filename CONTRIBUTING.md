# Contributing

## Principles

- Keep core stable.
- Put framework-specific behavior in modules.
- Prefer small focused modules.
- Avoid duplicating existing rules.
- Add examples where they reduce ambiguity.

## Adding a Module

1. Create `modules/<module-name>/SKILL.md`.
2. Follow `docs/module-authoring.md`.
3. Add routing instructions to `core/module-router/SKILL.md`.
4. Add usage examples if needed.

## Review Criteria

A contribution should be:

- clear
- reusable
- framework-aware
- compatible with existing FAOS rules
- safe for Codex Desktop usage
