# Design Review

Review frontend changes for architecture, design-system, accessibility, responsive behavior, and module compliance.

## Review Areas

- design token usage
- component reuse
- API quality
- accessibility
- responsive behavior
- styling consistency
- framework/module consistency
- documentation sync
- Storybook coverage
- visual regression risk

## Output Format

```text
FRONTEND DESIGN REVIEW
Summary:
Active Modules:
Risk Level: low | medium | high
Required Fixes:
Suggested Improvements:
Documentation Updates:
Storybook Updates:
Visual Regression Notes:
Final Recommendation: approve | approve with changes | revise
```
