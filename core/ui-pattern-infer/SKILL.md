# UI Pattern Infer

## Purpose

Map frontend requests to reusable UI patterns before implementation.

## Pattern Categories

- Action
- Form Input
- Navigation
- Overlay
- Feedback
- Data Display
- Layout
- Empty State
- Error State
- Loading State
- Data Visualization Container

## Output Format

```text
UI PATTERN INFERENCE

Pattern:
Intent:
Base Components:
Variants:
States:
Accessibility Model:
Responsive Behavior:
Token Dependencies:
Active Modules:
Documentation Updates:
```

## Rules

- Infer pattern before implementation.
- Prefer composition over one-off UI.
- Prefer existing components.
- Avoid visual-only names.
