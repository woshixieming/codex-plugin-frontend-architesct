# Design Review

## Purpose

Review frontend changes for architectural, design-system, accessibility, responsive, and module compliance.

## Review Areas

- design token usage
- component reuse
- API quality
- accessibility
- responsive behavior
- styling consistency
- framework/module consistency
- documentation sync
- Storybook coverage
- visual regression risk

## Output Format

```text
FRONTEND DESIGN REVIEW

Summary:
Active Modules:
Risk Level: low | medium | high

Required Fixes:
1.

Suggested Improvements:
1.

Documentation Updates:
1.

Storybook Updates:
1.

Visual Regression Notes:
1.

Final Recommendation:
approve | approve with changes | revise
```

## Rules

- Be concrete.
- Separate required fixes from suggestions.
- Avoid subjective comments unless tied to documented rules.
- Prefer minimal changes.
