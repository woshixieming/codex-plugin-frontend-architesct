# Storybook Planner

## Purpose

Plan Storybook coverage for reusable components.

## Required Stories

For reusable components, plan stories for:

- default
- variants
- sizes
- disabled
- loading
- error
- responsive
- accessibility
- theme variations if applicable

## Output Format

```text
STORYBOOK PLAN

Component:
Active Modules:
Stories Needed:
Missing Stories:
Story File Path:
Controls:
Accessibility Notes:
Visual Regression Coverage:
```

## Rules

- Follow existing Storybook conventions.
- Do not introduce Storybook unless requested or already present.
- Focus on coverage that improves review and regression testing.
